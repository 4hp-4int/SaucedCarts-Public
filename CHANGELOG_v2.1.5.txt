SaucedCarts v2.1.5
Released: 2026-04-26
====================

A bundle of MP-transfer fixes, durability quality-of-life, a couple of
flashlight install bugs, plus an opt-in BETA preview of the upcoming
corpse-storage feature.


BUG FIXES
---------

* Cart <-> shelf / freezer / counter / barbecue / wardrobe transfers
  could duplicate or silently no-op on dedicated MP. Fixed.

* Cart -> equipped backpack/satchel/holster sometimes deposited into
  your main inventory instead of the bag. Fixed for clients on v2.1.5.
  (Pre-v2.1.5 clients still mis-route this one direction with no dupe.
  Fully quitting PZ to desktop and relaunching loads the new code.)

* Cart inventory panel didn't refresh after transfers to/from an
  equipped cart. Fixed.

* Server-to-client broadcasts (cart visual sync, etc.) silently
  dropped on first session. Fixed.

* Single-player grapple lock — "spin and push but can't walk" after
  loading a corpse with a cart equipped. Fixed.

* "Grab Body" right-click on a fully-decomposed corpse inside a cart
  used to flicker the body in then out again with no feedback. Now
  silent-drops with the same halo as the drag-to-ground path.

* V hotkey while dragging a corpse (with corpse storage disabled) now
  halos "Drop the body before picking up a cart" instead of queueing
  a bugged pickup action.

* Flashlight install rejected valid attachment materials (Zip Ties,
  Rope, Adhesive Tape). Fixed — should now correctly accept any of
  Duct Tape, Zip Ties, Adhesive Tape, Rope, or Twine.


DURABILITY UX
-------------

Three small quality-of-life improvements addressing "my cart blew up
out of nowhere" reports:

* Threshold halos at 50% / 25% / 10% damage:
    "The cart is starting to creak under the load"
    "This cart is getting pretty beat up..."
    "This cart is about to fail! I need to repair or unload NOW."
  Each halo only fires once per damage cycle (no spam); repairing
  resets the cycle.

* Repair menu shows the material requirement inline, e.g. "Repair
  Cart (have Scrap Metal)" or "Repair Cart (need Scrap Metal)".


BETA OPT-IN: CORPSE STORAGE
---------------------------

A WIP feature ships in this build but is DISABLED BY DEFAULT.
One sandbox toggle: "Enable Corpse Storage [BETA]". When enabled:

  * Grapple a corpse, then right-click the cart -> "Load Corpse into
    <cart>" (or use the V hotkey while grappling). The corpse goes
    into the cart's inventory with full state preserved (clothes,
    rot stage, attached items).

  * Corpses age while stored. Past the sandbox HoursForCorpseRemoval
    threshold (vanilla despawn boundary), unloading silent-drops
    with a "Corpse fully decomposed" halo — same as if vanilla had
    despawned the body on the ground.

  * Cart-full / no-cart / decomposed paths halo immediately, even
    in MP race scenarios.

Off by default while the MP sync layer gets more polish. Public
release in a future version.


COMPATIBILITY
-------------

Save-safe. No ModData schema changes. Safe to upgrade mid-save.
